# Cyberpunk RED — React-приложение для управления персонажами

## Описание проекта

Это веб-приложение для создания и управления персонажами настольной ролевой игры **Cyberpunk RED**. Проект построен на современном стеке технологий с использованием React и предоставляет удобный интерфейс для работы с листами персонажей.

---

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Фреймворк | React 19.2.0 |
| Сборщик | Vite 6.3.5 |
| Роутинг | React Router DOM 7.6.0 |
| Язык | JavaScript (ES6+) |

## Функциональные возможности

### 1. Управление персонажами
- Создание, редактирование и удаление персонажей
- Переключение между персонажами
- Экспорт всех персонажей в JSON-файл
- Импорт персонажей из JSON-файла

### 2. Лист персонажа (Character Sheet)
- Просмотр и редактирование всех характеристик
- Сохранение данных в localStorage

### 3. Дополнительные разделы
- **Implants** — управление имплантами
- **Mobs** — управление NPC и противниками
- **Notes** — заметки и записи
- **Scripts** — скрипты и макросы

### 4. Бросок кубиков
- Встроенный диалог для броска игровых кубиков

## Хранение данных

Приложение использует **localStorage** браузера для персистентности данных:

- `characters` — массив всех персонажей (профили)
- `currentCharacterId` — ID выбранного персонажа
- `character_{id}_characterData` — данные конкретного персонажа

### Структура данных персонажа:
```javascript
{
  id: "char_1234567890",
  name: "Имя персонажа",
  description: "Описание",
  createdAt: 1234567890000,
  updatedAt: 1234567890000
}
```

## Особенности React-реализации

### 1. Компонентный подход
Весь интерфейс разбит на переиспользуемые компоненты:
- `Layout` — обёртка с навигацией для всех страниц
- `DiceDialog` — модальное окно броска кубиков
- Страницы как отдельные компоненты (`Characters`, `CharacterSheet`, etc.)

### 2. Управление состоянием через Context API
`CharacterContext` предоставляет глобальное состояние:
- Список всех персонажей
- Текущий выбранный персонаж
- Данные текущего персонажа
- Методы для CRUD-операций

```javascript
// Пример использования в любом компоненте:
const { characters, addCharacter, selectCharacter } = useCharacter()
```

### 3. Декларативный роутинг
Навигация между страницами через React Router:
```javascript
<Routes>
  <Route path="/" element={<Layout />}>
    <Route index element={<Navigate to="/characters" />} />
    <Route path="characters" element={<Characters />} />
    <Route path="sheet" element={<CharacterSheet />} />
    ...
  </Route>
</Routes>
```

### 4. Хуки для работы с состоянием
- `useState` — локальное состояние компонентов
- `useEffect` — побочные эффекты (сохранение в localStorage)
- `useCallback` — мемоизация функций
- `useContext` — доступ к глобальному контексту
- `useNavigate`, `useLocation` — навигация

### 5. Условный рендеринг
Динамическое отображение элементов в зависимости от состояния:
```javascript
{characters.length === 0 ? (
  <div>Нет персонажей</div>
) : (
  characters.map(char => <CharacterItem key={char.id} {...char} />)
)}
```

## Преимущества React перед чистым JS + HTML

### 1. **Компонентная архитектура**
| React | Чистый JS + HTML |
|-------|------------------|
| Переиспользуемые компоненты с собственным состоянием | Повторяющийся код, дублирование логики |
| Изолированная логика каждого компонента | Глобальная область видимости, конфликты имён |
| Лёгкое тестирование компонентов | Сложное тестирование спагетти-кода |

### 2. **Декларативный UI**
| React | Чистый JS + HTML |
|-------|------------------|
| UI описывается как функция от состояния: `UI = f(state)` | Ручное обновление DOM через `getElementById`, `innerHTML` |
| Автоматическая синхронизация состояния и UI | Ручное управление DOM при каждом изменении |
| Предсказуемое поведение | Легко пропустить обновление элемента |

### 3. **Виртуальный DOM**
| React | Чистый JS + HTML |
|-------|------------------|
| Эффективное обновление только изменённых узлов | Прямые манипуляции с реальным DOM (медленно) |
| Оптимизация перерисовок | Частые полные перерисовки при сложных изменениях |
| Высокая производительность при частых обновлениях | Падение производительности на больших приложениях |

### 4. **Однонаправленный поток данных**
| React | Чистый JS + HTML |
|-------|------------------|
| Данные передаются сверху вниз через props | Данные могут изменяться в любом месте |
| Предсказуемое состояние приложения | Сложно отследить, где изменились данные |
| Упрощённая отладка | Трудно воспроизвести баги |

### 5. **Хуки и управление состоянием**
| React | Чистый JS + HTML |
|-------|------------------|
| Встроенные хуки (`useState`, `useEffect`, `useContext`) | Самописные системы управления состоянием |
| Context API для глобального состояния | Глобальные переменные или сложные паттерны (Observer, Pub/Sub) |
| Мемоизация через `useCallback`, `useMemo` | Ручная оптимизация вычислений |

### 6. **Экосистема и инструменты**
| React | Чистый JS + HTML |
|-------|------------------|
| React Router для навигации | Ручная реализация роутинга через `window.location` |
| DevTools для отладки компонентов | Стандартные браузерные инструменты |
| Горячая перезагрузка (HMR) через Vite | Ручная перезагрузка страницы |
| TypeScript поддержка из коробки | Ручная настройка типов |

### 7. **Масштабируемость**
| React | Чистый JS + HTML |
|-------|------------------|
| Лёгкое добавление новых функций через компоненты | Код разрастается, становится неуправляемым |
| Чёткая структура проекта | Отсутствие стандартов структуры |
| Рефакторинг без поломки других частей | Изменения влияют на всю кодовую базу |

### 8. **Современный синтаксис**
```javascript
// React (JSX) — читаемо и понятно:
return (
  <div className="character-list">
    {characters.map(char => (
      <CharacterItem key={char.id} {...char} />
    ))}
  </div>
)

// Чистый JS — многословно и сложнее:
const list = document.createElement('div')
list.className = 'character-list'
characters.forEach(char => {
  const item = document.createElement('div')
  item.dataset.id = char.id
  item.textContent = char.name
  list.appendChild(item)
})
container.appendChild(list)
```

## Запуск проекта


## Вывод

Данный проект демонстрирует преимущества современного React-подхода:
- **Чистая архитектура** с разделением на компоненты, контексты и страницы
- **Эффективное управление состоянием** через Context API
- **Декларативный код**, который легко читать и поддерживать
- **Автоматическая оптимизация** через виртуальный DOM
- **Готовая экосистема** для роутинга, сборки и разработки

По сравнению с чистым JS + HTML, React позволяет создавать сложные интерактивные приложения с меньшим количеством кода, лучшей структурой и более высокой поддерживаемостью.
